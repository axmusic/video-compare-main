Loading State Control

You can customize the loading experience or disable it entirely using data attributes.

1. Customizable Loading Screen

Wrap your own spinner or text inside uevc-loading-wrapper. This will disappear once videos are ready.

Initializing Media... 
<div class="uevc-slider-container" data-show-loading="true">
    <div class="uevc-loading-wrapper">
        <div class="uevc-loading">
            <i class="fas fa-spinner fa-spin"></i> Initializing Media...
        </div>
    </div>
    <!-- videos here -->
</div>

2. Debug Loading Mode

Use data-debug-loading="true" to keep the loading screen visible indefinitely for styling purposes.

Debug Mode Active
<div class="uevc-slider-container" data-debug-loading="true">
    <div class="uevc-loading-wrapper">
        <div class="uevc-loading">Debug Mode Active</div>
    </div>
</div>

3. Loading Screen Disabled

Use data-show-loading="false" to completely prevent ANY loading screen from appearing. 

<div class="uevc-slider-container" data-show-loading="false">
    <!-- No loading wrapper needed -->
    <video>...</video>
</div>

Playback Modes

Control when the videos start playing.

1. Auto (Default)

Videos play automatically and loop.

Loading
<div class="uevc-slider-container" data-play-mode="auto">...</div>

2. Start on Interaction

Videos start paused and begin playing when first hovered or touched.

Loading
<div class="uevc-slider-container" data-play-mode="interaction">...</div>

3. Only Play While Interacting (Manual)

Videos only play when the mouse is over the container or while touching it.

Loading
<div class="uevc-slider-container" data-play-mode="manual">...</div>

Performance Optimization

Automatically pause videos when they are not in the viewport to save resources (CPU/GPU).

<div class="uevc-slider-container" data-viewport-pause="true">...</div>
